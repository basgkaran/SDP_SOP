Overview

The Service Catalog helps standardize and publish services such as onboarding, software installation, access requests, etc., with custom forms, approvals, SLAs, and task workflows.

Navigation – Full Paths

Configure Catalog: Admin → Service Catalog

Add New Service: Admin → Service Catalog → Add Service

Organize Categories: Admin → Service Catalog → Categories

Request View: Home → Request → New → Select from Service Catalog

Key Configurations

Service Categories: Group services logically (e.g., HR Requests, Hardware, Software).

Service Templates: Custom forms per service, with dynamic fields and logic.

Approvals: Define multi-level approvals (e.g., Manager → Dept. Head).

Fulfillment Tasks: Set task workflows for services with dependencies and owners.

SLA Mapping: Assign SLA per service template and urgency level.

Technician Assignment: Auto-assign based on category/site/priority.

Best Practices

Group services into 5–8 logical categories: Improves usability and clarity for end users.

Use descriptive names: “Request Laptop (Permanent)” instead of “Laptop Req”.

Use conditional approvals and dynamic fields: Simplifies workflows.

Define clear task ownership: Prevent delays or fulfillment errors.

End User View

Users see categorized services, easy-to-fill forms, request status updates, and get email notifications for approvals and fulfillment.

Technician/Admin View

Technicians can manage requests, monitor SLA/task status, and maintain templates and automation rules. Admins can run reports on fulfillment trends.
