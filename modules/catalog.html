Overview

The Service Catalog provides a structured way to offer IT and business services to end users, with form-based requests, approvals, SLAs, and task automation.

Navigation

Configure Catalog: Admin → Service Catalog

Add New Service: Admin → Service Catalog → Add Service

Organize Categories: Admin → Service Catalog → Categories

Request View: Home → Request → New → Select from Service Catalog

Key Configurations

Service Categories: Group services (e.g., HR Requests, Software, Hardware)
Service Templates: Custom request forms with dynamic fields and approvals
Approvals: Multi-level approval logic based on request data
Fulfillment Tasks: Assign sequenced or parallel tasks to complete service
SLA Mapping: Set service-level targets based on type, urgency, etc.

Best Practices

Use clear naming: e.g., “New Laptop Request (Permanent)”
Group templates logically: 5–8 categories work best
Restrict access: Use department/site conditions to show relevant templates
Version control: Deactivate old templates before replacing them

End User View

End users see services by category, fill structured forms, receive approval status updates, and track service delivery easily via portal or email.

Technician/Admin View

Technicians and admins configure services, approve requests, manage fulfillment tasks, and monitor catalog usage and SLAs for optimization.
