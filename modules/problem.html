Overview

Problem Management helps identify recurring incidents, conduct root cause analysis (RCA), and eliminate underlying issues to improve service quality and reduce unplanned downtime.

Navigation – Full Paths

Problem List: Problems → All Problems

Link Incidents: Problems → View → Associated Requests

Root Cause Templates: Admin → Problem Management → Root Cause Templates

Problem Settings: Admin → Problem Management → Settings

Key Configurations

Problem Templates: Standardized layout for logging, assessing, and resolving problems. Includes RCA, impact, and workaround fields.

Root Cause Analysis (RCA): Methods like 5 Whys, Fishbone Diagrams can be documented in structured RCA fields.

Associated Incidents: Link related incidents to understand the scale and frequency.

Workarounds: Define temporary solutions that can be shared with technicians while the problem is under investigation.

Closure Code: Include cause category, resolution summary, and lessons learned for historical tracking.

Best Practices

Proactively analyze recurring incidents: Set a threshold (e.g., 5 similar tickets in 30 days) to trigger a new problem record.

Involve SMEs during RCA: Technical teams should collaborate to avoid misdiagnosing problems.

Document workarounds clearly: Helps technicians resolve related incidents quickly before final resolution is available.

Risks & Mitigation

Unlinked incidents: Hinders pattern recognition. Train technicians to tag related problems.

Inadequate RCA: Fixes symptoms, not causes. Use structured formats and cross-team RCA reviews.

Problems left open indefinitely: Create SLA or review cadence for problem aging and closure.

Technician/Admin View

Technicians use the problem view to log details, link incidents, perform RCA, update workarounds, and suggest changes. Admins configure templates and track aging reports.
