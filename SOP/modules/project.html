Overview

The Project Management module in ServiceDesk Plus enables structured planning and delivery of IT or cross-functional projects, with milestone tracking, task assignment, dependencies, and collaboration tools.

Navigation – Full Paths

Project Dashboard: Projects → All Projects

Create Project: Projects → New Project

Project Template: Admin → Project Management → Templates

Task Types: Admin → Project Management → Task Types

Role Setup: Admin → Project Management → Project Roles

Key Configurations

Project Templates: Define repeatable project structures with pre-defined milestones and task layouts.

Milestones: High-level checkpoints with deadlines and dependencies. Use them to segment projects into manageable phases.

Tasks & Subtasks: Break work into actionable items, assign owners, track % completion, and set due dates.

Dependencies: Define task relationships (Start-to-Start, Finish-to-Start) to enforce sequence.

Roles & Permissions: Define who can create, edit, or view projects/tasks.

Best Practices

Use templates for repeatable projects: Saves time and ensures consistency.

Assign project owners: One SPOC per project improves accountability.

Use color-coded statuses: (e.g., On Track, Delayed, Completed) to visually monitor progress.

Risks & Mitigation

No task dependencies defined: May cause planning confusion. Always use dependency mapping for clarity.

Poor project scope: Leads to scope creep. Define objectives and constraints clearly at the start.

Lack of progress tracking: May result in last-minute delays. Enforce regular % completion updates.

Technician/Admin View

Technicians view tasks assigned to them, update statuses, and comment on blockers. Admins oversee all projects, update templates, and run reports on project health.
