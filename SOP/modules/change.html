Overview

The Change Management module enables organizations to request, assess, approve, and implement changes in a controlled and auditable manner. It supports workflows, CAB reviews, risk assessments, and rollbacks.

Navigation – Full Paths

Change Dashboard: Changes → All Changes

Change Template: Admin → Change Management → Templates

Change Workflow: Admin → Change Management → Change Workflow

Stage Settings: Admin → Change Management → Stages and Status

CAB Configuration: Admin → Change Management → Change Advisory Board

Key Configurations

Change Templates: Define structured change request forms based on change type (Standard, Emergency, Normal, Major).

Change Workflow: Visual drag-and-drop builder to map stages (Submission → Review → Approval → Implementation → Post-review).

CAB Roles: Define CAB members, their responsibilities, and schedule meetings with automated reminders.

Approval Phases: Multi-stage approvals with conditional logic based on risk, impact, or change category.

Rollback Plan: Include fallback steps for each implementation in case the change fails.

Best Practices

Use a Standard Change Template: For frequently approved, low-risk changes (e.g., password reset policy updates).

Integrate with CMDB: Always link changes to affected Configuration Items for better impact analysis.

Track Metrics: Use KPIs like Change Success Rate, Emergency Change Count, and CAB Review Turnaround.

Risks & Mitigation

Skipped Approvals: Can cause unauthorized changes. Use mandatory stage settings.

Incomplete Implementation Plans: Risk of failure. Templates should mandate pre-checks and post-implementation steps.

Overused Emergency Changes: Indicates poor planning. Track root causes and push back into planned changes.

Technician/Admin View

Technicians can raise, track, and execute changes. Admins define workflows, configure CABs, and monitor compliance across the change lifecycle.
