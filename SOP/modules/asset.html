Overview

The Asset Management module allows organizations to maintain a centralized inventory of hardware, software, and non-IT assets. It supports discovery, life cycle tracking, licensing, and asset relationships.

Key Configurations

Asset Types: Classify assets as IT (laptops, desktops, routers) or Non-IT (furniture, vehicles). Helps with visibility and reporting.

Asset Discovery: Use agent-based or network scan for auto-discovery. Recommended to deploy agents for laptops/desktops across sites.

Ownership Mapping: Assign users to assets to track movement and accountability.

Software License Compliance: Track installations vs purchased count. Set alerts for overuse or expiry.

Contracts & Warranty: Upload contracts and map to assets. Get alerts 30/60/90 days prior to expiry.

Best Practices

Define Naming Conventions: Use prefixes like LAP-101, SRV-502 for traceability.

Schedule Scans: Weekly scans help identify changes and avoid shadow IT risks.

Use Location/Site Mapping: For better access control and tracking per region.

Tag End-of-Life Assets: Automate workflows for decommissioning.

Risks and Mitigation

Duplicate Assets: May result from CSV import and scans. Use deduplication rules.

License Non-Compliance: Risk of audit failure. Always track installs vs entitlement.

Unassigned Assets: Poor accountability. Link users to each asset for traceability.

End-User View

End users may view assigned assets via the portal. Access is read-only and limited to their own items.

Technician/Admin View

Admins manage asset data, run reports, update license compliance, monitor warranties, and receive alerts. Technicians can use the asset info for faster ticket resolution.
