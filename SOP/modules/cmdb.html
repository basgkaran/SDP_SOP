Overview

The CMDB in ServiceDesk Plus helps document, map, and track all Configuration Items (CIs) such as servers, applications, and business services along with their interdependencies.

Navigation – Full Paths

CMDB Dashboard: CMDB → All CIs

Add CI Types: Admin → CMDB → CI Types

Add Relationships: Admin → CMDB → Relationship Types

Import CIs: CMDB → Import → From CSV

Dependency Map: CMDB → CI View → Dependency Map

Key Configurations

CI Types: Classify as Server, Network, Application, Database, etc. Define custom attributes for each type.

CI Attributes: Define technical and business attributes like IP, location, criticality, owner, and vendor.

Relationship Types: Define relationships like “Runs On”, “Depends On”, “Backed Up By” for mapping.

Dependency Mapping: Visualize CI relationships and service impact. Useful for RCA and Change Planning.

CI Lifecycle: Track CI statuses: In Use, In Maintenance, Retired. Align with change history and ticket associations.

Best Practices

Keep the CI scope limited: Start with core infrastructure and grow gradually.

Use automation: Pull CI data from discovery or integrations with asset tools like Endpoint Central.

Maintain clear relationships: Helps with RCA and understanding service impact during outages.

Tag changes to CIs: Ensure every Change/Incident is linked to a CI wherever applicable.

Risks & Mitigation

Too many CI Types: May cause clutter. Focus on core types with real IT impact first.

Missing relationships: Affects impact analysis. Encourage CI owners to update maps during changes.

Data Drift: CI details can become outdated. Sync CMDB with asset scans or scheduled reviews.

Technician View

Technicians can view CIs linked to tickets, assess risk/impact, and log updates. Dependency maps assist during outages and escalations.

Admin View

Admins create CI types, relationships, import CI data, and manage visual dependency graphs. They ensure data integrity and CI associations with changes/incidents.
