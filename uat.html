UAT & Go-Live - Phase Execution

Plan, test, and transition smoothly into live operations

UAT Preparation

Test Scenarios: Build UAT cases covering all request flows, SLAs, automation, approvals, assets, and service catalog items.

UAT Users: Assign testers from each key department (IT, HR, Facilities) and document roles for evaluation.

Checklists: Provide a dry-run checklist with expected outcomes and review points.

Feedback Capture: Use Excel or Google Forms to record issues and improvement suggestions.

Go-Live Readiness

Admin Lock: Freeze configuration changes once UAT is signed off to prevent regressions.

Data Cleanup: Remove test tickets, dummy assets, and stale users from UAT.

Final Sync: Import fresh requester/asset/user data into production environment.

Production Launch Communication: Notify all stakeholders with go-live date, what’s changing, and whom to contact for support.

Initial Post-Go-Live Monitoring

Daily Checks: Track SLAs, request volumes, auto-assignments, and escalations daily for the first 2 weeks.

Feedback Loop: Accept change requests with strict tracking (avoid scope creep in early weeks).

Weekly Status Calls: Monitor adoption KPIs and action fixes quickly.

Best Practices

Use a versioned release plan: Rollout in phases (e.g., Core modules first, then advanced features).

Always take backup exports: Save UAT configuration, templates, and user imports before switching to production.

Log early change requests: Keep a backlog of enhancements for version 2.0 instead of modifying live setup continuously.
